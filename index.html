Problem to product

폐기 직전 재고를 거래 가능한 시간 안에 판매 흐름으로 바꿉니다.

01

소상공인 재고 부담

마감 시간이 가까운 상품은 판매 가능해도 폐기되기 쉽고, 실시간 할인 판매를 운영할 도구가 부족합니다.

02

소비자 탐색 비용

사용자는 가까운 매장의 할인 상품과 픽업 가능 시간을 한 화면에서 비교하기 어렵습니다.

03

현장 픽업 검증

주문 승인 후 수령 여부를 명확히 확인해야 판매자 운영과 정산 흐름이 안정적으로 이어집니다.

Product flow

구매자와 판매자 앱을 하나의 주문 상태 흐름으로 연결했습니다.

Buyer App

  1. 탐색홈, 카테고리, 검색, 지도 마커
  2. 주문서버 장바구니 동기화, 시루 잔액 기반 결제 UI
  3. 픽업주문 상태 폴링, 픽업 코드, QR, 카운트다운, Tmap 길찾기
REST API + JWT

Seller App

  1. 등록메뉴 기반 상품 등록, 할인가, 수량, 픽업 시간
  2. 처리신규, 준비중, 픽업대기, 완료 상태 관리와 QR/코드 검증
  3. 운영매출 차트, 반경 기반 고객 알림, 정산 신청, Mock 시연

Key features

시연에서 바로 설명 가능한 기능 단위

지도 기반 탐색

Google Maps로 현재 위치와 주변 매장 마커를 표시하고, Tmap API로 보행 경로를 안내합니다.

주문 상태 전이

주문 생성부터 승인, 픽업 완료, 취소까지 구매자와 판매자 화면을 백엔드 상태로 동기화합니다.

픽업 검증

구매자 QR/픽업 코드를 판매자 앱에서 스캔하거나 직접 입력해 현장 수령을 확정합니다.

운영 대시보드

판매 상품, 주문 현황, 매출 분석, 인기 메뉴, 고객 알림, 정산 흐름을 판매자 앱에 모았습니다.

Quality & operations

로컬 시연부터 운영 점검까지 확인 가능한 개발 경로를 갖췄습니다.

시연 환경

구매자/판매자 앱은 Debug 로그인으로 백엔드 플로우를 빠르게 확인하고, 판매자 앱은 Mock 세션으로 API 없이도 핵심 화면을 시연할 수 있습니다.

릴리스 방어

Android 릴리스 빌드에서 기본 에뮬레이터 주소를 그대로 쓰지 않도록 차단하고, API 키와 서버 주소는 local.properties로 분리했습니다.

테스트와 문서

백엔드는 JUnit 5, Testcontainers, Jacoco, 미완성 Stub 검증 태스크와 Swagger/OpenAPI, Postman 컬렉션으로 회귀 확인 경로를 제공합니다.

운영 관측

Flyway 마이그레이션, Actuator, Micrometer, Prometheus, Grafana, Docker Compose, k6 테스트로 데이터와 성능 상태를 추적합니다.

Architecture

두 Android 앱이 하나의 Kotlin Spring Boot 4 API를 통해 거래 상태를 공유합니다.

Contribution

서비스 흐름을 기술 구현으로 연결한 지점을 포트폴리오 핵심으로 둡니다.

Backend design

인증, 매장, 상품, 장바구니, 주문, 찜, 리뷰, 알림, 정산 API를 역할별 컨트롤러와 서비스로 구성했습니다.

Transaction flow

구매자 주문 생성과 판매자 주문 처리를 하나의 상태 모델로 연결하고, 픽업 코드 검증으로 수령 완료를 확정했습니다.

Operational visibility

Actuator, Prometheus, Grafana 기반 모니터링과 Swagger/OpenAPI, Postman, k6로 개발 및 운영 확인 경로를 마련했습니다.
